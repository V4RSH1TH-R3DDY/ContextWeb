Hot-swap control: ContextOS can switch between Gemini Flash, Gemini Flash Lite, Groq fallback, and deterministic rules without changing the user flow. The UI shows which engine made the recommendation, how confident it was, and which fallback would run if AI is unavailable.

Built for production. Not a prototype.

Tech Stack

  • Language: Kotlin, Kotlin DSL Gradle
  • UI: Jetpack Compose + Material Design 3
  • DI: Hilt (Dagger)
  • Database: Room (SQLite) + KSP
  • Background: Foreground Service + WorkManager + AlarmManager
  • Serialization: kotlinx.serialization
  • Testing: JUnit, MockK, Robolectric, MockWebServer

Multi-Module Architecture

  • :app — Jetpack Compose UI, Navigation, ViewModels
  • :core:service — Foreground Service + OpenClaw Agent loop
  • :core:skills — Skill interface + 6 skill implementations
  • :core:memory — Routine, Preference, Location learning
  • :core:network — Calendar, Gmail, Drive, Maps API clients
  • :core:data — Room DB, DAOs, SituationModel domain classes

System Flow (15-min cycle)

1. SensorDataCollector → RawSensorData (≤3s)
2. SituationModelBuilder.build() incl. calendar + memory (≤4s)
3. OpenClawAgent.analyzeSituation() → SituationAnalysis (≤2s)
4. skill.shouldTrigger() × 6 skills (≤1s)
5. skill.execute() → SkillResult → ActionDispatcher (≤3s)
6. ActionLog write to Room DB + Notification
Total: 12s compute + 3s dispatch = <15s per cycle
